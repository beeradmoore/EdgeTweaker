Microsoft Edge Browser Policy Documentation EdgeProxyPacFailOpenEnabled | Microsoft Learn

Enable fail-open behavior for Proxy PAC on iOS

Supported versions

  • Windows: Not supported
  • macOS: Not supported
  • Android: Not supported
  • iOS: ≥ 144

Description

This policy controls whether Microsoft Edge for iOS allows network requests to proceed when the Proxy Auto-Config (PAC) file specified in EdgeProxyPacUrl can't be downloaded or evaluated.

If you enable this policy or don't configure it, network requests are allowed to proceed directly without using a proxy when the PAC file is unavailable. If you disable this policy, network requests are blocked when the PAC file can't be retrieved or processed, ensuring all traffic must go through the configured proxy.

Note: This policy takes effect only when EdgeProxyPacUrl is configured.

Supported features

  • Can be mandatory: Yes
  • Can be recommended: No
  • Dynamic Policy Refresh: Yes
  • Per Profile: Yes
  • Applies to a profile that is signed in with a Microsoft account: No

Data type

  • Boolean

iOS information and settings

  • Preference Key name: EdgeProxyPacFailOpenEnabled
  • Example value:

xml <true/>
