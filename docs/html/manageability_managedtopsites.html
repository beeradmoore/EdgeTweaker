Configure managed top sites on New Tab Page

Supported versions

  • Windows: Not supported
  • macOS: Not supported
  • Android: ≥ 146
  • iOS: ≥ 145

Description

This policy controls a list of managed top site shortcuts displayed on the New Tab Page in Microsoft Edge for iOS and Android.

If this policy is enabled, you can specify one or more top site shortcuts to display on the New Tab Page. Each shortcut requires the following fields:

  • name: The display name of the shortcut.
  • url: The URL of the site.

The shortcuts configured by this policy are managed and cannot be edited or deleted by users.

If this policy is disabled or not configured, users have full control over their top sites.

Supported features

  • Can be mandatory: Yes
  • Can be recommended: No
  • Dynamic Policy Refresh: Yes
  • Per Profile: Yes
  • Applies to a profile that is signed in with a Microsoft account: No

Data type

  • Dictionary

Android information and settings

  • Preference Key name: ManagedTopSites
  • Example value:

[{"name": "Outlook", "url": "https://outlook.office.com"}, {"name": "Microsoft", "url": "https://www.microsoft.com"}]

iOS information and settings

  • Preference Key name: ManagedTopSites
  • Example value:

xml <key>ManagedTopSites</key> <array> <dict> <key>name</key> <string>Outlook</string> <key>url</key> <string>https://outlook.office.com</string> </dict> <dict> <key>name</key> <string>Microsoft</string> <key>url</key> <string>https://www.microsoft.com</string> </dict> </array>
